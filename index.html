01 AI System
AI Expense Tracker
Problem:
Tools like YNAB and Copilot AI cost $10–15/month with limited intelligence. Users receive generic budget suggestions disconnected from their actual spending patterns. No free alternative exists that provides genuinely personalized, context-aware financial guidance.
Solution:
Open-source AI expense tracker using GPT-4 function calling to analyze transaction history, detect spending anomalies, generate personalized budget plans, and answer natural language financial queries — at zero recurring cost.
Python FastAPI GPT-4 API LangChain PostgreSQL Next.js Plaid API Redis Docker
↗ GitHub Repository ↗ Live Demo
Core Features
  • Natural language transaction queries ("How much did I spend on food in March?")
  • Anomaly detection — flags unusual charges and duplicate transactions
  • AI-generated monthly budget based on income + spending history
  • Bank integration via Plaid API (read-only, zero credential storage)
  • Smart categorization using embeddings + few-shot classification
  • Predictive monthly spend forecasting with confidence intervals
System Architecture
Plaid API → Transaction Ingestion Service
→ Preprocessing Pipeline (normalize, dedupe)
→ Vector Embeddings (pgvector)
→ FastAPI Backend + LLM Agent Layer
→ Next.js Dashboard
⬡ AI Integration

Model: GPT-4 with function calling + custom financial tools. The LLM acts as a reasoning layer over structured transaction data — it never directly accesses raw DB queries. Limitation: LLM accuracy degrades with 12+ months of sparse data; addressed via sliding window context.

Roadmap
  • Fine-tuned classifier for hyper-accurate category detection
  • Investment portfolio tracking integration
  • Multi-user household budget sharing
  • Local model option (Llama 3) for full privacy

Solution:

03 Systems
RAG Document Intelligence Engine
Problem:
Enterprise document search is broken — keyword search misses semantic context, and generic LLMs hallucinate when answering questions about proprietary documents. Organizations need an AI that accurately answers questions grounded in their own knowledge base.
Solution:
Production-grade RAG pipeline supporting PDF, DOCX, and web content ingestion. Hybrid search (dense + sparse), automatic chunking strategy selection, citation tracking, and confidence scoring on every answer.
Python LlamaIndex pgvector FastAPI OpenAI Celery Next.js
↗ GitHub Repository
Core Features
  • Hybrid retrieval: BM25 keyword search + dense vector similarity
  • Adaptive chunking based on document type and structure
  • Source citation with paragraph-level references in every answer
  • Confidence scoring — LLM abstains when certainty is below threshold
  • Async document ingestion pipeline via Celery + Redis
  • Multi-tenant namespace isolation per organization
Ingestion Pipeline
Document Upload
→ Parser (PDF/DOCX/HTML)
→ Chunker (semantic/fixed/sliding)
→ Embedder (text-embedding-3)
→ pgvector + BM25 index
→ Query → Hybrid Retrieval → LLM
⬡ AI Integration

Embedding: OpenAI text-embedding-3-large. Generation: GPT-4-turbo with strict grounding prompt — explicitly instructed to cite sources and refuse when context is insufficient. Hallucination rate reduced by 78% vs naive RAG baseline in internal testing.

Roadmap
  • Table and chart extraction from PDFs
  • Multi-hop reasoning across documents
  • Local embedding model support (BGE-M3)

System Design Breakdowns

REST API Design

Resource-oriented endpoints with consistent response envelopes. All mutations are idempotent. Versioned via URL prefix (/v1/) to support backwards compatibility.

# Consistent response envelope { "status": "success", "data": { ... }, "meta": { "page": 1, "total": 42 }, "errors": null }

Database Schema

PostgreSQL with pgvector extension. Normalized schema with strategic denormalization for read performance. TimescaleDB for time-series health metrics.

Scalability Strategy

Horizontal scaling via stateless API design. Redis for session state and rate limiting. Celery task queue for async LLM calls. CDN-cached static assets.

# Rate limiting + caching layer @cache(ttl=3600, key="user:{id}") @rate_limit(rpm=60, burst=10) async def get_insights(user_id): return await llm.analyze(...)
🔒
Security Architecture

JWT with short expiry + refresh token rotation. Row-level security in PostgreSQL for multi-tenant isolation. All LLM inputs sanitized against prompt injection.

# Prompt injection guard def sanitize_input(text: str) → str: # Strip system-role patterns return re.sub( r"(system:|<|INST>)", "", text )
⚡
LLM Performance Optimization

Semantic caching for similar queries reduces LLM calls by ~40%. Streaming responses for perceived speed. Prompt compression via LLMLingua for long contexts.

# Semantic cache lookup hit = await cache.find_similar( query_embedding, threshold=0.95 ) if hit: return hit.response else: return await llm.call(...)
🧪
LLM Evaluation Pipeline

Automated eval suite using RAGAS for RAG quality, custom rubrics for agent reasoning. Every deployment runs regression tests on 50 golden Q&A pairs.

# RAGAS evaluation metrics metrics = [ faithfulness, # 0.92 answer_relevancy, # 0.88 context_recall, # 0.85 context_precision # 0.91 ]
